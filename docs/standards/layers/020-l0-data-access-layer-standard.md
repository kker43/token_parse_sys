# Layer Standard 020: L0 Data Access 数据访问层

## 1. 层级定义

L0 负责通过稳定契约读取已发布数据资产。

它回答：

```text
如何从已批准的数据产品中读取指定标的和日期的数据？
```

## 2. 所有权和职责

本层拥有：

- 数据资产目录读取。
- 数据源适配器。
- 行级读取接口。
- 查询参数和来源依赖记录。

本层不拥有：

- 数据采集修复。
- 指标计算。
- 策略语义。

## 3. 输入和输出

输入：

- `DataAssetCatalog`。
- 数据库/文件连接配置。
- 查询参数。

输出：

- 标准化行数据。
- 查询依赖信息。

## 4. 边界和禁止事项

禁止：

- 绕过 DataAsset 契约直接读未知表。
- 在适配器里做策略计算。
- 静默吞掉缺失或质量错误。

## 5. 生产准入

新增 L0 适配器必须：

- 有契约测试。
- 有连接配置样例。
- 能返回稳定字段。
- 不暴露敏感凭据。

## 6. 生产准出

L0 输出给 L1 前必须：

- 字段名符合契约。
- 主键和日期字段完整。
- 质量状态可检查。

## 7. 迭代增加规则

新增数据产品读取能力应按产品批量增加，不要为单个策略写临时 SQL。

## 8. 淘汰和降级规则

当上游资产废弃、schema 不兼容或质量长期失败时，L0 读取能力必须降级或废弃。

## 9. 质量和异常监控

监控：

- 查询失败率。
- 空结果率。
- schema mismatch。
- 读取延迟。

## 10. 复杂性控制

L0 只做读取，不做 pivot、打标签或策略判断。

## 11. Agent 检查清单

- [ ] 读取对象已在 DataAsset 中注册。
- [ ] 查询字段受控。
- [ ] 没有策略逻辑。
- [ ] 失败可解释。
