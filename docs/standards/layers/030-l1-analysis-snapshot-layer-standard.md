# Layer Standard 030: L1 Analysis Snapshot 分析快照层

## 1. 层级定义

L1 把 L0 读取到的多源事实数据组织成单标的、单日期的分析快照。

它回答：

```text
某只股票在某个日期，研究和策略可以观察到哪些稳定特征？
```

## 2. 所有权和职责

本层拥有：

- `AnalysisSnapshot`。
- 特征命名和依赖追踪。
- 多源数据合并。
- 快照版本。

本层不拥有：

- 原语判断。
- 标签语义。
- 策略召回。

## 3. 输入和输出

输入：

- L0 行数据。
- DataAsset 契约。
- stock_code 和 snapshot_date。

输出：

- `AnalysisSnapshot`。
- feature map。
- dependencies。

## 4. 边界和禁止事项

禁止：

- 在 L1 里判断“突破”“强势”“买点”。
- 直接读取外部服务。
- 合并未来日期数据。

## 5. 生产准入

新增 L1 特征前必须：

- 明确来源字段。
- 明确日期语义。
- 明确缺失处理。
- 有构建测试。

## 6. 生产准出

快照可被 L2 消费前必须：

- 必需字段存在。
- dependencies 完整。
- analysis_version 明确。
- run_id 可追溯。

## 7. 迭代增加规则

研究需要的新特征可以先由 research job 临算；只有多场景复用后才进入 L1 正式快照。

## 8. 淘汰和降级规则

如果特征来源失效、语义重复、长期无人依赖，应废弃或降级为 research-only。

## 9. 质量和异常监控

监控：

- 快照构建成功率。
- 必需特征缺失率。
- source dependency 完整率。
- 未来数据检查。

## 10. 复杂性控制

L1 只组织可观察特征，不扩大为策略特征仓库。

## 11. Agent 检查清单

- [ ] 特征来源可追溯。
- [ ] 日期没有未来泄漏。
- [ ] 没有 L2/L3 语义。
- [ ] 已补构建测试。
