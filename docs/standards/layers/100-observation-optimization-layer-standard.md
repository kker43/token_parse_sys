# Layer Standard 100: Observation And Optimization 观察复盘优化层

## 1. 层级定义

观察复盘层跟踪 test_tracking 或生产策略在后续真实日期中的表现。

它回答：

```text
策略上线观察后，真实召回和后续表现是否符合预期？
```

## 2. 所有权和职责

本层拥有：

- ObservationRecord。
- ReviewFinding。
- 策略表现复盘。
- 新版本建议。

本层不拥有：

- 直接修改 L2/L3/L4。
- 静默替换生产策略。

## 3. 输入和输出

输入：

- L5 信号。
- 后续价格表现。
- 策略预期。

输出：

- 观察记录。
- 失败案例。
- 优化建议。
- 新候选版本。

## 4. 边界和禁止事项

禁止：

- 根据短期观察直接改生产版本。
- 只记录成功案例。
- 删除失败样本。

## 5. 生产准入

进入观察前必须：

- 策略状态为 test_tracking 或 active_production。
- 有观察周期。
- 有评价指标。

## 6. 生产准出

观察结论可推动迭代前必须：

- 样本数足够。
- 成功和失败都记录。
- 与回测预期比较。

## 7. 迭代增加规则

优化建议应批量进入新 candidate version，不直接覆盖旧策略。

## 8. 淘汰和降级规则

若观察表现持续低于阈值，应降级策略状态或停止信号生成。

## 9. 质量和异常监控

监控：

- 信号后 5/10/20 日收益。
- 回撤。
- 策略命中数量。
- 回测和观察偏差。

## 10. 复杂性控制

观察层只生成复盘证据，不承担新策略研发的全部逻辑。

## 11. Agent 检查清单

- [ ] 观察对象状态正确。
- [ ] 成功失败样本都有。
- [ ] 没有静默改策略。
