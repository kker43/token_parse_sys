# Layer Standard 060: L3 Labels 语义标签层

## 1. 层级定义

L3 把多个 L2 原语组合成可复用业务语义。

它回答：

```text
这些原子状态组合起来，代表什么业务标签？
```

## 2. 所有权和职责

本层拥有：

- `LabelDefinition`。
- `LabelSnapshot`。
- 标签 registry。
- 标签版本。

本层不拥有：

- 买入卖出。
- 排序和组合权重。
- 回测逻辑。

## 3. 输入和输出

输入：

- L2 primitive 输出。

输出：

- 语义标签。
- matched/score 等标签字段。

## 4. 边界和禁止事项

禁止：

- 直接消费 L1 原始字段。
- 直接读取数据源。
- 表达交易动作。
- L3 依赖另一个 L3 形成隐式链条。

## 5. 生产准入

L3 标签进入 approved 前必须：

- 所有依赖 L2 已批准或 production_candidate。
- 有正样本和反例。
- 语义可人工解释。
- 有回测或观察证据。

## 6. 生产准出

L3 可被 L4 使用前必须：

- label_id 唯一。
- 版本稳定。
- 输出字段稳定。
- 适用范围明确。

## 7. 迭代增加规则

先在 research 层验证，再批量注册同一形态族标签。

## 8. 淘汰和降级规则

如果标签无法区分正反样本、长期无人依赖或下游收益不稳定，应废弃。

## 9. 质量和异常监控

监控：

- 标签命中率。
- 命中率漂移。
- 下游策略表现。
- 失败案例。

## 10. 复杂性控制

标签组合不宜过深。复杂策略准备态可以放到 `composite_setup`，但仍必须由 L2 直接组合。

## 11. Agent 检查清单

- [ ] 只消费 L2。
- [ ] 不表达交易动作。
- [ ] 有正反样本。
- [ ] 有 registry。
