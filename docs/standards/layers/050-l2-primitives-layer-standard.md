# Layer Standard 050: L2 Primitives 原子状态层

## 1. 层级定义

L2 负责单一维度的原子状态判断。

它回答：

```text
某个单一状态是否成立？
```

## 2. 所有权和职责

本层拥有：

- `PrimitiveDefinition`。
- 纯函数计算。
- 原语 registry。
- 原语版本。

本层不拥有：

- 完整形态。
- 策略动作。
- 多标签组合。

## 3. 输入和输出

输入：

- L1 AnalysisSnapshot。

输出：

- boolean、numeric 或 category 原子状态。

## 4. 边界和禁止事项

禁止：

- 读取 MySQL、文件或 API。
- 表达买卖动作。
- 引用策略名。
- 把多个业务语义揉成一个原语。

## 5. 生产准入

L2 原语进入 approved 前必须：

- 输入字段已稳定。
- 计算函数纯净。
- 阈值有样本证据。
- 有单测。
- 有 registry 记录。

## 6. 生产准出

L2 可被 L3 使用前必须：

- primitive_id 唯一。
- version 明确。
- missing_policy 明确。
- 输出类型稳定。

## 7. 迭代增加规则

新原语优先从多个 L3 候选中抽象出来，不因为单个策略直接批准。

新增 L2 前必须先执行因子复用检查：

- 是否已有同义原语。
- 是否只有窗口不同。
- 是否只有日线/周线/月线周期不同。
- 是否可以复用已有基础指标或同类滚动统计口径。
- 是否只是把多个原子状态组合成一个策略语义。

如果只是窗口不同，例如 `max_drawdown_60d`、`max_drawdown_120d`、`weekly_max_drawdown_26w`，应优先归入同一个 `rolling_max_drawdown(close, window)` 口径族，而不是新增策略专属原语。

## 8. 淘汰和降级规则

原语若语义重复、阈值失效、输入不再稳定，应 deprecated。

## 9. 质量和异常监控

监控：

- 原语计算失败率。
- true/false 分布漂移。
- 缺失输入比例。
- 下游依赖数量。

## 10. 复杂性控制

每个 L2 只回答一个问题。复杂语义放到 L3。

## 11. Agent 检查清单

- [ ] 只消费 L1。
- [ ] 只表达一个状态。
- [ ] 有测试。
- [ ] 有 registry。
