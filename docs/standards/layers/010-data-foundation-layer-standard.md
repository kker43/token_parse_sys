# Layer Standard 010: Data Foundation 数据基础层

## 1. 层级定义

Data Foundation 负责事实数据和基础技术指标生产。

它回答：

```text
某个市场、某个标的、某个日期，哪些事实数据和基础指标是可用、可信、可追溯的？
```

## 2. 所有权和职责

本层拥有：

- 事实类行情、财务、资产元数据。
- 基础技术指标，例如 MA、区间高低点、成交额均值、波动率。
- 数据质量状态。
- `DataAsset` 目录导出。

本层不拥有：

- L2 原语语义。
- L3 标签语义。
- L4 策略逻辑。
- 买卖信号和策略评价。

## 3. 输入和输出

输入：

- 外部数据源。
- 已批准的数据源配置。
- 生产日期和市场范围。

输出：

- `pub_*` 数据产品。
- 数据质量状态。
- 数据资产契约。
- 基础指标注册表。

## 4. 边界和禁止事项

禁止：

- 输出“强势股”“买点”“突破关注”等经验语义。
- 为单个策略临时修改生产口径。
- 在没有质量监控时进入例行调度。

## 5. 生产准入

基础指标进入例行生产前必须满足：

- 计算口径白盒、无未来函数。
- 至少 2 个研究 workflow 复用，或 1 个高优先级策略长期依赖。
- 有字段来源、窗口、缺失策略。
- 计算成本可控。
- 有质量监控和异常告警。

## 6. 生产准出

数据产品可以被 L0/L1 消费前必须满足：

- `status=ready`。
- `quality_level in (pass, warning)`。
- 记录数达到最低预期。
- 数据日期不晚于消费日期。

## 7. 迭代增加规则

研究阶段的新指标先保持：

```text
research_only 或 candidate
```

当多个策略复用且通过准入评审后，再批量纳入基础生产。不要为单个样本随手新增例行指标。

## 8. 淘汰和降级规则

必须降级或废弃：

- 口径被证明有未来函数。
- 数据源不稳定且无替代。
- 长期无人依赖。
- 与更稳定指标重复。

## 9. 质量和异常监控

必须监控：

- 记录数。
- 日期连续性。
- 空值率。
- 异常极值。
- 数据延迟。
- 与历史分布偏离。

异常处理：

- 阻断下游正式生产。
- 输出 `warning` 或 `failed`。
- 保留错误原因和 run id。

## 10. 复杂性控制

基础层优先批量迭代，不做碎片化策略定制。

新增指标必须证明：

- 可复用。
- 易解释。
- 低成本。
- 可监控。

## 11. Agent 检查清单

- [ ] 这是事实或基础指标，不是经验语义。
- [ ] 有 DataAsset 契约。
- [ ] 有质量检查。
- [ ] 有缺失策略。
- [ ] 没有未来函数。
