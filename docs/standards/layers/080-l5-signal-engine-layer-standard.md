# Layer Standard 080: L5 Signal Engine 信号生成层

## 1. 层级定义

L5 把 L4 策略应用到候选池，生成可跟踪的策略信号。

它回答：

```text
在某个信号日期，哪些股票被策略召回、过滤、排序？
```

## 2. 所有权和职责

本层拥有：

- `StrategySignal`。
- 召回原因。
- 过滤通过项。
- 排名和分数。

本层不拥有：

- 策略定义。
- 回测评价。
- 交易执行。

## 3. 输入和输出

输入：

- L4 StrategyDSL。
- L3 LabelSnapshot。
- 候选池。

输出：

- 信号列表。
- 排名。
- 解释字段。

## 4. 边界和禁止事项

禁止：

- 改写策略规则。
- 引入未注册标签。
- 静默发布质量失败信号。

## 5. 生产准入

L5 例行生成前必须：

- L4 策略状态允许。
- L3 标签快照质量通过。
- 候选池完整。

## 6. 生产准出

信号可进入观察前必须：

- 记录 strategy_id/version。
- 记录 signal_date。
- 记录召回原因。
- 记录 warning checks。

## 7. 迭代增加规则

新增排序字段或解释字段应以策略版本变更形式批量迭代。

## 8. 淘汰和降级规则

如果信号长期为空、异常暴增、依赖标签失效，应暂停生成。

## 9. 质量和异常监控

监控：

- 每日信号数量。
- 空信号天数。
- 召回暴增暴减。
- 依赖缺失。

## 10. 复杂性控制

L5 只执行策略，不重写策略。

## 11. Agent 检查清单

- [ ] 策略状态允许生成信号。
- [ ] 标签快照完整。
- [ ] 输出可解释。
