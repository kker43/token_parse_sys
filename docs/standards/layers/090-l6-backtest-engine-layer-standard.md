# Layer Standard 090: L6 Backtest Engine 回测层

## 1. 层级定义

L6 对 L4/L5 产出的策略或事件进行可复现评估。

它回答：

```text
策略在给定口径下的收益、胜率、回撤和样本质量如何？
```

## 2. 所有权和职责

本层拥有：

- `BacktestResult`。
- 事件回测。
- 持有期收益。
- 最大回撤。
- 胜率和样本数。

本层不拥有：

- 策略定义。
- 标签定义。
- 生产审批。

## 3. 输入和输出

输入：

- 策略事件或信号。
- 价格序列。
- 回测口径。

输出：

- BacktestResult。
- trade details。
- skipped events。

## 4. 边界和禁止事项

禁止：

- 在回测中改变策略信号。
- 使用未来数据生成信号。
- 忽略样本不足。

## 5. 生产准入

新增回测引擎或口径前必须：

- 执行假设明确。
- 入场/出场价格明确。
- 手续费/滑点策略明确或显式说明未计。
- 有单测。

## 6. 生产准出

回测结果可用于策略准入前必须：

- sample_size 明确。
- failure_cases 明确。
- period 明确。
- benchmark 明确。
- metrics 完整。

## 7. 迭代增加规则

新增指标或持有期应批量比较，不要只挑最优结果报告。

## 8. 淘汰和降级规则

若发现未来函数、价格字段错误、样本选择偏差，回测结果必须作废。

## 9. 质量和异常监控

监控：

- skipped events。
- 样本数。
- 极端收益。
- 最大回撤。
- 回测版本。

## 10. 复杂性控制

先做白盒事件回测，再扩展组合回测。复杂撮合逻辑必须单独版本化。

## 11. Agent 检查清单

- [ ] 交易假设明确。
- [ ] 没有未来函数。
- [ ] skipped events 已记录。
- [ ] 结果能回灌 L4 准入。
