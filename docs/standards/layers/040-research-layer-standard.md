# Layer Standard 040: Research 研究编排层

## 1. 层级定义

Research 负责从样本、截图、人工观察和临时计算中沉淀候选经验。

它回答：

```text
某个形态或现象是否值得沉淀为 L2/L3/L4 候选？
```

## 2. 所有权和职责

本层拥有：

- PatternCase。
- FactorObservation。
- PrimitiveCandidate。
- LabelCandidate。
- StrategyCandidate。
- 研究临算工具。
- 样本证据和失败案例。

本层不拥有：

- 正式生产指标。
- 已批准 L2/L3/L4。
- 例行信号发布。

## 3. 输入和输出

输入：

- L1 snapshot。
- 样本图、日期、人工观察。
- 临时计算指标。

输出：

- 候选原语。
- 候选标签。
- L4 draft 策略。
- 回测和观察证据。
- production promotion evidence。

## 4. 边界和禁止事项

禁止：

- 把研究临算直接当作基础生产。
- 静默批准策略。
- 只用正样本不记录反例。

## 5. 生产准入

研究 artifact 进入候选注册前必须：

- 有清晰中文解释。
- 有计算口径。
- 有样本证据。
- 有依赖字段。

## 6. 生产准出

研究产物交给 L2/L3/L4 前必须：

- 指明目标层级。
- 指明状态。
- 指明缺口。
- 通过 production promotion review 或人工审批。

## 7. 迭代增加规则

研究层允许小步快跑。

推荐节奏：

```text
单样本理解 -> 小样本验证 -> 反例补充 -> 回测 -> 候选注册
```

## 8. 淘汰和降级规则

如果形态只在单个样本有效、阈值不稳定、反例过多，应保留为研究记录，不进入生产。

## 9. 质量和异常监控

监控：

- 正样本数量。
- 反例数量。
- 回测样本数。
- 阈值稳定性。
- 失败案例覆盖。

## 10. 复杂性控制

研究层可以探索复杂组合，但进入 L2/L3/L4 前必须拆解成清晰小口径。

## 11. Agent 检查清单

- [ ] 这是候选研究，不是正式生产。
- [ ] 正反样本都有记录。
- [ ] 有回测或观察证据。
- [ ] 已明确下一层目标。
