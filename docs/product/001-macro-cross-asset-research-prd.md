# PRD 001：宏观大类资产投研系统

## 1. 背景和目标

本 PRD 定义 Stock Lobster 向宏观大类资产跟踪扩展时的产品形态。它不是把本系统改造成事实数据生产系统，而是在现有分层基础上消费外部数据契约，形成可复现的宏观分析快照、资产趋势判断、异动提醒和下钻研究入口。

目标用户需要每天回答四类问题：

1. 主要大类资产现在处于什么趋势？
2. 哪些资产或宏观变量出现了异常变化？
3. 异动背后的宏观解释是否有足够证据？
4. 下一步应该下钻到哪些市场、行业、期限结构、链上指标或股票池？

## 2. 产品定位

产品名称暂定为 `Macro Cross-Asset Research`。

它是 Stock Lobster 的宏观研究扩展，不替代 A 股策略系统。A 股仍是第一阶段策略执行主线；宏观大类资产模块提供更高层的市场环境、风险偏好、流动性和跨资产联动上下文。

系统输出的是研究结论、趋势标签、异动提醒和下钻任务，不直接输出交易指令。若未来需要把宏观结论用于 A 股策略，只能通过已批准的 L3 标签和 L4 StrategyDSL 引用。

## 3. 资产范围

MVP 覆盖：

| 资产组 | 第一批对象 | 典型问题 |
| --- | --- | --- |
| 贵金属 | 黄金、白银 | 避险、实际利率、美元方向是否支持趋势？ |
| 能源 | WTI、Brent、天然气 | 供需、库存、期限结构是否支持油价方向？ |
| 大宗商品 | 铜、铁矿、农产品、商品综合指数 | 全球增长、通胀、库存周期是否共振？ |
| Crypto | BTC、ETH、稳定币流动性、链上活跃度 | 风险偏好、流动性、链上资金是否一致？ |
| 股市 | A 股宽基、A 股风格、A 股行业、港股、美股、全球 DM/EM | 风险资产趋势、风格轮动、区域强弱如何？ |
| 美债 | 2Y、10Y、30Y、期限利差、实际利率、通胀预期 | 利率路径、增长预期、衰退交易是否变化？ |
| 中国国债 | 1Y、10Y、30Y、期限利差、信用利差代理 | 国内增长、政策和流动性环境如何？ |

后续扩展：

- 外汇：DXY、USDCNH、EURUSD、JPY、主要商品货币。
- 信用：美高收益利差、投资级利差、国内信用利差。
- 波动率：VIX、MOVE、A 股波动率代理。
- 资金面：美元流动性、央行资产负债表、逆回购、DR007 等。

## 4. 用户场景

### 4.1 每日宏观资产看板

用户打开系统后应看到：

- 大类资产总览：趋势方向、趋势强度、异动状态、置信度。
- 宏观环境：增长、通胀、流动性、风险偏好、美元、利率、信用。
- 今日变化：价格异动、收益率曲线变化、商品库存或链上数据异常。
- 需要关注的下钻任务：例如“黄金上涨但实际利率未确认，需要检查美元和央行购金数据”。

### 4.2 异动提醒

系统应在资产或宏观变量出现异常时生成提醒：

- 单资产异动：BTC 单日涨幅超阈值且成交量放大。
- 期限结构异动：2Y-10Y 美债利差快速陡峭化。
- 跨资产背离：黄金上涨但美元和实际利率同时上行。
- 宏观数据冲击：CPI、就业、PMI、央行会议后资产响应异常。

提醒必须说明触发规则、输入数据、历史分位、同类历史场景和置信度。

### 4.3 下钻分析

从总览或异动进入下钻：

1. 资产层：趋势、波动、成交量、期限结构、资金流。
2. 驱动层：增长、通胀、利率、美元、流动性、风险偏好。
3. 比较层：同组资产强弱、跨资产联动、历史相似阶段。
4. 影响层：对 A 股宽基、风格、行业和策略候选池的影响。

### 4.4 结论审阅

系统可以生成候选结论，但不能把 Agent 解释当成事实。每条结论必须有：

- 结论类型：趋势、异动、宏观 regime、风险提示、下钻建议。
- 证据列表：使用了哪些 DataAsset、窗口、指标和标签。
- 反证列表：哪些指标不支持该结论。
- 置信度分解：数据质量、信号一致性、跨资产一致性、历史验证、时效性。
- 审批状态：`draft`、`reviewed`、`approved_for_research_use`、`rejected`。

## 5. 非目标

MVP 不做：

- 事实数据采集、清洗、修复或改写。
- 自动交易、仓位建议或订单执行。
- 新闻爬取和未经验证的消息事实生产。
- 无来源的宏观解释生成。
- 直接让 Agent 生成正式信号、回测结果或已批准策略。

## 6. 产品对象

| 对象 | 含义 | 所属边界 |
| --- | --- | --- |
| `MacroDataAsset` | 外部宏观或市场数据契约 | L0 |
| `AssetUniverse` | 资产组和可跟踪标的列表 | L0/L4 配置 |
| `MacroAnalysisSnapshot` | 某日宏观环境快照 | L1 |
| `AssetStateSnapshot` | 某资产在某日的趋势、波动、估值或期限结构视图 | L1 |
| `CrossAssetSnapshot` | 跨资产联动和相对强弱视图 | L1 |
| `MacroPrimitive` | 作用于快照的纯函数，例如趋势、分位、背离 | L2 |
| `MacroLabelSnapshot` | 确定性宏观和资产标签 | L3 |
| `MacroResearchConclusion` | 研究结论候选和审阅状态 | research |
| `DrilldownTask` | 下钻任务，不是交易信号 | app/research |

## 7. 输出形态

MVP 输出三类报告：

1. `daily_macro_cross_asset_brief`：每日总览。
2. `cross_asset_anomaly_report`：异动解释和反证。
3. `asset_drilldown_report`：单资产或资产组下钻。

每份报告必须可追溯到 `run_id`、快照版本、数据资产版本和规则版本。

## 8. MVP 里程碑

### P0：产品和契约草案

- 完成 PRD、系统 spec、第一批 DataAsset 示例。
- 明确资产组、数据源候选、置信结论路径和边界。

### P1：L0/L1 数据契约和快照

- 为 FRED、EIA、美国财政部、Crypto 数据源和现有 A 股数据产品建立 DataAsset 配置。
- 生成合成或只读样例 `MacroAnalysisSnapshot`、`AssetStateSnapshot` 和 `CrossAssetSnapshot`。

### P2：L2/L3 趋势和异动标签

- 实现趋势、分位、波动、背离和事件响应原语。
- 生成可复现的宏观和资产标签快照。

### P3：研究报告和提醒

- 基于 L3 标签生成研究结论候选。
- 建立置信度分解和反证展示。
- 用户审阅后才允许结论进入可复用研究库。

### P4：A 股下钻联动

- 将宏观标签作为 A 股策略的外部环境上下文。
- 只通过已批准标签进入 L4，不允许策略直接读取宏观原始数据。

## 9. 成功指标

- 每日简报可以在固定时间生成，且所有字段可追溯。
- 每条异动都有确定性触发规则和反证列表。
- 结论置信度不是单一 Agent 打分，而是由可解释证据分解构成。
- 宏观结论不会绕过 L3/L4 直接影响正式信号。
- 用户能从总览下钻到资产、驱动因素和 A 股影响链路。
